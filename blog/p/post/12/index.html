🐚 Microtag 🐚

[Voltar]


Algumas das coisas que eu uso como escapismo da minha vida de merda são a programação e as computarias como hobby. Eu gosto especialmente de criar coisas com shell script. Quando eu estou com tédio ou com vontade de procrastinar, abro o vim e começo a escrever algum script para resolver algum problema que eu tenho, ou simplesmente porque deu vontade de tentar criar alguma coisa específica.

Desta vez eu inventei de tentar fazer um script para gerar galerias estáticas. Ele gera um site no estilo booru, mas read-only. Sem opções para subir arquivos e nem nada que requeira um back-end, pois o objetivo é criar um site estático que possa ser subido no neocities, github, etc. Algo simples, mas que funcione.

Para ver um exemplo, clique aqui. Eu estou usando neste blog uma versão levemente modificada do microtag para organizar todos os aquivos que subo no site.

Funcionamento

Não tem muitas opções. Tudo que o script faz é importar para a galeria todos os arquivos no diretório IMPORT_DIR, que por padrão é /tmp/microtag-import. Quando se usa a opção --import, ele vai nesse diretório e busca por arquivos *.tags. Esse é um arquivo de texto com tags separadas por espaços. Existindo, ele busca pelo arquivo que o *.tags faz referência, que deve ter o mesmo nome, mas sem o .tags no final.

$ ls /tmp/microtag-import/
078a49f7fdd99a6bf38542a3a05ec883.webp
078a49f7fdd99a6bf38542a3a05ec883.webp.tags

Ou seja, o script já espera que você tenha colocado lá no diretório um arquivo a ser importado e um arquivo *.tags de mesmo nome com as tags. Como você vai fazer isso? Não sei. Ou manualmente, ou com algum script. Eu uso um script.

$ ./microtag --import
[Looking for new files...]
[078a49f7fd...ec883.webp / 1.8M]
  |  Converting to webp and copying...
  |  Took 3s.
  |  Creating thumb...
  |  Took 0s.
  |  1.8M → 536K/webp
$ ./microtag --build
[Building]
  |  Templates copied.
  |  Took 0s.
  |  JS database and tags uploaded.
  |  Took 0s.

A opção --import importa os arquivos para dentro do site e atualiza um banco de dados sqlite com os metadados deles. Suporta imagens, gifs e vídeos. Imagens e gifs são convertidos para webp por ser um formato muito mais eficiente. Vídeos são só copiados. A opção --build atualiza os arquivos dos sites com os arquivos do diretório ./templates e atualiza também o banco de dados JSON do site final.

Outras coisas

Ele tem a opção --select-by-tag que retorna a lista de md5s de todos os arquivos com aquela tag. A lista pode ser pipada nas opções --delete ou --update para mass delete ou mass changes.

$ ./microtag --select-by-tag webp
c3fe5f6b260922245f9e65495692b6e1
$ ./microtag --select-by-tag webp | ./microtag --delete
[Deleting]
  |  Deleted: c3fe5f6b260922245f9e65495692b6e1
  |  576K → 36K
  |  Took 0s.

Tem paginação e suporta pesquisar multiplas tags e também pesquisa por exclusão usando o sinal - antes da tag.

Pesquisa: foo -bar
# Todos os arquivos possuindo tag foo, exceto aqueles possuindo tag bar.

Nada demais, faz jus ao nome micro e é bastante simples. Apenas algo que quis compartilhar porque gostei do resultado. Funciona bem. Pelo menos na minha máquina.
